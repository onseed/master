주석 및 네이밍 규칙

HTML
일반적인 페이지 : 산출물ID.html
Ajax DOM 컨텐츠(step case) : 산출물ID_step01.html
Case별 페이지 : 산출물ID_case01.html
더보기 클릭시 Ajax로 추가되는 리스트 페이지 : 산출물ID_more.html
레이아웃 공통(인클루드) : gnb_include.html, footer_include.html 등.
레이어 팝업: 산출물ID_pop.html
주석 가이드
수정영역 주석 : <-- 날짜_내용 -->, <-- //날짜_내용 -->
수정내역은 페이지내 body태그 바로 위에 기재(최근 수정일이 앞으로) : <-- 수정일 150925/150921 -->
CMS영역 주석(줄 맨앞으로) : <-- CMS -->, <-- //CMS -->
AJAX영역 주석(줄 맨앞으로) : <-- AJAX -->, <-- //AJAX -->
개발요구사항 주석 : <-- 개발참고사항 : 메세지 -->, <-- //개발참고사항 : 메세지 -->

HTML 작성

  • Style 및 script 요소의 type 속성 생략
  • JS파일 인클루드는 하단에 작성: jquery 플랫폼 및 초기화 js는 예외
  • 프로젝트 내 모든 네임은 소문자 사용.
  • Table 에 summary 대신 caption을 이용하여 정보 제공.
  • z-index는 50까지만 사용
    header: 50 , contents:1-10, footer: 20~30
  • 상하 컨텐츠간의 간격은 확장을 고려하여 하단 컨텐츠 기준으로 간격 정의
    ex) top-margin,top-padding으로 정의
  • 단독 여백 클래스 사용 지양.
    ml, mt, mgt10, mgt20등 여백만 지정되어 있는 클래스는 수정 발생시 html 수정까지 발생하게 되므로 되도록 사용 안함.
  • html 태그 내 속성 순서(권장):
    'src(href)-class-name-id-title-alt-인라인스타일' 순
  • 인라인 태그(a/span) 중첩 형태중 가장 하위태그는 span
    ex) <a href="#" title=""> <span>링크</span> </a> (O)
    ex) <span> <a href="" title=""> </a> </span> (X)
  • 인풋박스 삭제 버튼 (IE8,9) 확인 필요

CSS

CSS 작성

  • '_' 로 구분하여 선언 ex) .guide_list
  • '#' id로 스타일 설정 불가 (id는 스크립트에서만 사용)
  • 'ui_'클래스 접두어는 UI 스크립트 class로 사용 지양.(권장)
    ex) .ui_box (X)
  • '_01'로 작성 불가.
    ex) .tbl_list1 (X)
  • 버튼 - btn_*
    테이블 - tbl_*
    탭 - tab_*
  • CSS3의 자식선택자, 인접형제선택자, A태그외 의사클래스 사용 절대 불가.(html 구조 수정 이슈 발생.)
    최대한 크로스브라우징이 되지않는 CSS3 자제.
    ex) ol > li (X)
    ex) p + p (X)
    ex) p:hover, span:hover, button:hover (X)
  • 의사 클래스의 경우 A태그 이외에 사용이 불가하므로 클래스로 정의. 대부분 스크립트 구현.
    ex) 마우스오버/탭: .active
    ex) 활성화 상태: .on
  • '_01'로 작성 불가. ex) .tbl_list1
  • 버튼 - btn_*
    테이블 - tbl_*
    탭 - tab_*
  • 'ui_'클래스 접두어는 스크립트 관련 class로 사용 지양.(권장)

Image

  • '_' 로 구분하여 선언
    ex) 이름1_이름2_01, 이름1_이름2_02.png
  • 별도 블릿 및 라인 이미지 사용시 아래 형식 네이밍.
    bl_색상_모양_사이즈.png
    ex) bl_gray_square_2x2.png
    ex) line_light_gray_19x1.png

  • 유형별 이미지는 sprite 기법으로 작업.
  • btn_* : 버튼
    ico_* : 아이콘
    bl_* : 블릿
    bg_* : 배경
    img_* : 페이지명_이미지성격: 기타 이미지

    line_* : 라인

접근성

접근성 고려

  • 한페이지에 유일한 버튼명일 경우 버튼 타이들은 기재하지 않음
    동일 버튼명이 한페이지 내에 다수 존재시 버튼 타이틀 기재
  • alt="" 값은 200자 제한. 그 이상일 경우 IR 기법 사용. IR기법시 hide 클래스 .hide {display: inline-block; position: absolute; z-index: -10; width: 0 !important; height: 0 !important; overflow: hidden !important; opacity: 0 !important; filter: alpha(opacity=0); -ms-filter: 'progid:DXImageTransform.Microsoft.Alpha(opacity=0)'; font-size: medium; line-height: normal; word-break: break-all;}
  • target="_blank"외 새 창은 title="새 창"
  • value와 placeholder속성은 스크린리더에서 지원하지 않으므로 반드시 타이틀과 함께 기재.
  • 윈도우 팝업의 페이지 타이틀은 팝업창-제목 형식으로 기재
  • 아이프레임 사용시 타이틀 사용.
  • 레이어 팝업은 반드시 "contents" 영역밖으로 처리
    (레이어 팝업 모듈은 탭키등 포커스 이동 이슈로 "contents" 영역은 aria-hidden 처리됨)
  • A link: 단순 URL을 가지고 있는 페이지 이동 혹은 새창일 경우
    Button: 정보성(데이터)을 포함하고 있는 경우, 페이지가 이동 혹은 새창일 때
  • 작업 후 검수요청 전 자가검수 (탭컨트롤/접근성확인 tool 확인)

모바일 웹접근성 주의사항

안드로이드는 크롬 에뮬레이터로 테스트 가능하니
모바일 접근성은 ios기준으로 테스트 해야함.
링크 텍스트를 font-size:0으로 숨기는 경우 iOS_VoiceOver 초점이 접근불가.
<a>를 <button>으로 교체하거나 <a>에 role="button"을 추가.
pc나 안드로이드는 div에 tabindex나 focus가 가능하나 ios는 불가능.
레이어팝업 오픈시 초점을 레이어팝업내 버튼에 두도록 함.

반응형

반응형 작성

  • 모바일과 PC는 동일한 DOM 구조 (ONE Source)
  • 소규모 프로젝트는 반응형 미디어쿼리 처리
  • 페이지수가 많거나 분리해야 하는 경우는 채널형, 적응형으로 명시하여 아래와 같이 처리
  • 모바일 CSS는 따로 존재하며 PC웹과 상이한 부분은 별도 스타일을 지정.
  • 모바일 CSS명은 PC웹과 동일하며 "_mobile"을 구분자 추가
  • html 소스내 PC웹과 모바일 CSS를 둘다 import, 쌍으로 존재. <link rel="stylesheet" type="text/css" href="css/guide.css"> <!-- pc용 --> <link rel="stylesheet" type="text/css" href="css/guide_mobile.css"> <!-- mobile용 -->
  • 개발에서 분리하거나, html단에서 분기처리

지원 해상도

  • 모바일은 별도 미디어쿼리로 처리
  • 모바일은 태블릿 사이즈(768)까지 지원(PC용 CSS의 .w768 스타일과 유사)
  • PC화면 해상도에 따른 구분 클래스(로드시만 적용)
    900 이상: w1
    768 이상: w1 w2
    413 이상: m12 m2
    413 이하: m12 m1
